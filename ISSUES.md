# ISSUES.md — Reguli pentru GitHub Issues

Orice modificare de cod în acest repository **MUST** fie asociată cu un GitHub Issue.

## Reguli

1. Înainte de a face orice modificare, creează un Issue care descrie problema sau task-ul.
2. Commit-ul **MUST** conține referința la Issue în mesaj: `"Descriere (#123)"`.
3. După ce modificările sunt publicate, închide Issue-ul cu un comentariu care sumarizează ce s-a făcut.
4. Issue-urile se numerotează secvențial: #1, #2, #3, etc.
